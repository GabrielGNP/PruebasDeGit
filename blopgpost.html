Curso Profesional de Git y GitHub

En este blog se van a describir los distintos conceptos aprendidos sobre Git y GitHub en el curso Profesional de Git y GitHub de Platzi dado por Freddy Vega

Modificacion de texto agregado para comprobar Ramas





$ git --version : permite mostrar en la consola la version de git instalada en la maquina



$ git : escribe en pantalla informacion breve sobre cada uno de los comandos que se pueden utilizar con git



$ pwd : escribe en la consola la direccion actual del directorio. La carpeta actual donde se está direccionando. Escribe la direccion completa



$ cd : (change directory) permite cambiar la carpeta referenciada, cambia de directorio de trabajo. $ cd c/ Selecciona carpeta c (o disco c). Si se escribe $ cd .. entonces volverá al directorio padre. $ cd . hace referencia a la carpeta actual.



$ ls: Escribe en pantalla los archivos y carpetas dentro del directorio referenciado.
Si se utiliza el argumento -l entonces mostrará los archivos y carpetas listados uno debajo de otro. $ ls -l
Si se utiliza el argumento -a entonces tambien mostrará los archivos ocultos dentro del directorio. $ ls -a
Pueden combinarse los arbumentos -a y -l para mostrar todos los elementos, incluidos los ocultos, listandolos uno debajo de otro. -al o -la, el orden es indiferente. $ ls -al



$ clear : Permite limpiar la consola de todas las instrucciones dadas. Tambien puede hacerse CTRL + L para limpiar la pantalla sin necesidad de ejecutar ninguna instruccion.



$ mkdir NombreCarpeta : Permite crear un nuevo directorio dentro del directorio de trabajo. NombreCarpeta será el nombre que se le dará a la carpeta.



$ touch NombreArchivo.Extension : Permite crear un nuevo archivo dentro del directorio de trabajo. NombreArchivo será el nombre del archivo y .Extension será la extension del archivo



$ cat NombreArchivo.Extension : Muestra el contenido del archivo al cual se le está haciendo referencia con la instruccion.



$ history : Muestra el historial de instrucciones que se ha ejecutado durante toda la ejecucion de la consola. Cuando se cierra la consola, el historial se vacía. El historial se mantiene aunque se realize un $ clear .



$ !Número : Ejecuta la instruccion con el número del historial. Si en el historial hay una ejecucion realizada con el número 79, al realizar $ !79 entonces se ejecutará la misma instruccion que se encuentre en el historial con el número 79.



$ rm NombreArchivo.Extension : Permite borrar el archivo con el nombre dado. Si no se maneja con cuidado puede llegar a borrar incluso todo un disco.



--help : Es un argumento que permite ver una descripción mucho más completa de la instrucción a la que se le agregue este argumento. escribir $ git es igual que $ git --help . Por ejemplo se puede hutilizar $ rm --help para conocer el funcionamiento del $ rm .
